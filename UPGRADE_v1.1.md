## Что нужно сделать

### 1. Установить новую зависимость

В папке проекта:

```bash
npx expo install react-native-reanimated
```

### 2. Заменить файлы

Скачай новый архив и замени этими файлами у себя:

- `App.js` — добавлен `GestureHandlerRootView`
- `babel.config.js` — **НОВЫЙ ФАЙЛ**, без него reanimated не заработает
- `src/components/WorldMapSvg.js` — полностью переписан на reanimated
- `src/components/CountryRegionMap.js` — поддержка TopoJSON и проекции Albers USA
- `src/screens/CountryDetailScreen.js` — передаёт `countryId` в карту
- `src/data/countries.js` — новый URL для США

### 3. ⚠️ ОБЯЗАТЕЛЬНО: полный перезапуск с очисткой кэша

Babel-плагин reanimated применяется при компиляции. Если не сбросить кэш, он не подхватится и получишь ошибку вида `Reanimated 2 failed to create a worklet`.

**Останови текущий `expo start` (Ctrl+C) и запусти:**

```bash
npx expo start --tunnel --clear
```

Флаг `--clear` обязательно нужен при первом запуске после этих изменений.

На телефоне в Expo Go тоже лучше полностью закрыть приложение и открыть заново.

## Как пользоваться новой картой

- **Один палец**: двигать карту

- **Двойной тап**: быстрый зум / сброс
- **Кнопка ⊙ справа вверху**: сброс к исходному виду
- **Тап по стране**: отметить

---

## Если что-то пошло не так

**Ошибка `Reanimated 2 failed to create a worklet`:**
→ Не сбросил кэш. Выполни `npx expo start --tunnel --clear`

**Ошибка `Can't find variable: __reanimatedModuleProxy`:**
→ Не установил или плохо установил reanimated. Сделай `npx expo install react-native-reanimated` ещё раз.

**Белая карта США:**
→ Плохой интернет когда загружалась. Закрой детальный экран, открой снова.

**Карта мира всё ещё тормозит:**
→ Это особенность Expo Go (он медленнее чем production build). На скомпилированном приложении будет ещё быстрее.
