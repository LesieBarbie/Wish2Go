## Що потрібно зробити

### 1. Встановити нову залежність для сповіщень

У папці проекту:

```bash
npx expo install expo-notifications
```

### 2. Замінити файли з архіву

Розпакуй `travel-app-v2.0.zip` і заміни свою папку на цю. Або просто заміни такі файли:

- `App.js`
- `src/context/TravelContext.js`
- `src/data/countries.js` (багато нових країн-регіонів та переклад)
- `src/data/achievements.js` (переклад)
- `src/screens/MapScreen.js`
- `src/screens/ProfileScreen.js`
- `src/screens/AchievementsScreen.js`
- `src/screens/CountryDetailScreen.js`
- `src/components/CountryRegionMap.js` (підтримка geoBoundaries + кращий пошук назв регіонів)
- `src/components/AchievementToast.js` — **НОВИЙ ФАЙЛ**
- `src/utils/notifications.js` — **НОВИЙ ФАЙЛ**

### 3. Скинути кеш і перезапустити

```bash
npx expo start --tunnel --clear
```

На телефоні закрий Expo Go повністю (змахни з списку), відкрий заново, скануй QR.

## Як перевірити що сповіщення працюють

1. При першому відкритті додаток попросить дозвіл на сповіщення — **дозволь**
2. Натисни на будь-яку країну на мапі → отримаєш ачівку «Перший крок» → побачиш:
   - Зелену картку зверху екрана з анімацією
   - Системне сповіщення у шторці iPhone (свайпни зверху щоб перевірити)
3. Якщо скинеш усе через кнопку «Скинути все» у профілі — ачівки знову можна буде отримати, бо скидання чистить історію сповіщень.

## Про push-сповіщення в Expo Go

На iPhone в Expo Go працюють **локальні** сповіщення (такі, які сам додаток надсилає телефону — саме те, що нам треба для ачівок). Віддалені push (коли сервер надсилає) потребують окремого build'у — але нам це поки не потрібно.

**Якщо сповіщення не приходять:**
- Перевір налаштування iPhone: Налаштування → Сповіщення → Expo Go → увімкни
- Закрий і відкрий додаток заново

## Якщо в якоїсь країни карта регіонів не завантажується

Це може статися якщо джерело даних недоступне або у нього немає потрібного поля `name`. Напиши який саме — я або знайду інше джерело, або додам правильний ключ для назви регіону. Можливі варіанти назв полів: `name`, `NAME`, `NAME_1`, `shapeName`, `nom`, `NAME_EN`.

### Швейцарія — якщо не працюватиме

Можливо, джерело `click_that_hood` тимчасово недоступне. Якщо так — скажи, переключу на geoBoundaries (він точно надійний).
